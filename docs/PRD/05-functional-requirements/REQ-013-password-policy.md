# REQ-013 密码策略管理

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **估时** | 3 人天 |
| **关联用户故事** | US-013 |

**背景：** 密码是用户认证的第一道防线，需要完善的密码策略来防止弱密码、密码泄露和暴力破解风险，满足企业级安全合规要求。

**目标：**

- 支持密码强度校验，防止弱密码
- 支持密码历史记录，防止重复使用旧密码
- 支持密码过期策略，定期强制更换
- 支持密码错误锁定策略
- 满足 SOC2 合规要求

**功能描述：**

### 1. 密码强度校验

支持以下强度规则配置：

| 规则 | 说明 | 默认要求 |
|------|------|----------|
| 最小长度 | 密码最少字符数 | ≥ 8 位 |
| 大写字母 | 至少包含大写字母 | ≥ 1 个 |
| 小写字母 | 至少包含小写字母 | ≥ 1 个 |
| 数字 | 至少包含数字 | ≥ 1 个 |
| 特殊字符 | 至少包含特殊字符 (!@#$%^&*) | ≥ 1 个 |
| 连续字符 | 禁止连续递增/递减字符 | 禁止 123/abc |
| 重复字符 | 禁止同一字符重复 | ≤ 3 个连续 |
| 字典词检查 | 禁止常见弱密码 | 禁止 password/admin 等 |

强度等级配置：
- `LOW`: 仅最小长度
- `MEDIUM`: 最小长度 + 大小写 + 数字（默认）
- `HIGH`: 最小长度 + 大小写 + 数字 + 特殊字符
- `CUSTOM`: 自定义规则组合

### 2. 密码历史记录

1. 记录用户最近 N 次密码的哈希值
2. 修改密码时，检查是否与历史密码重复
3. 默认保留最近 5 次密码记录
4. 可配置禁止重复的周期（如 6 个月内不重复）

### 3. 密码过期策略

1. 支持设置密码有效期（默认 90 天）
2. 密码即将过期前提醒用户更换：
   - 过期前 7 天：登录时提示
   - 过期前 3 天：登录时强提醒
   - 已过期：强制跳转密码修改页
3. 支持密码永不过期配置（适用于服务账号）
4. 密码修改后，过期时间重新计算

### 4. 密码错误锁定策略

1. 连续密码错误达到阈值，锁定账号
2. 默认配置：
   - 5 次错误：锁定 15 分钟
   - 10 次错误：锁定 1 小时
   - 20 次错误：锁定 24 小时
3. 支持管理员手动解锁
4. 支持邮箱/短信验证自助解锁

### 5. 密码修改流程

1. 用户输入当前密码和新密码
2. 校验当前密码正确性
3. 校验新密码符合强度要求
4. 校验新密码不在历史记录中
5. 更新密码并记录历史

### 6. 密码重置流程（忘记密码）

1. 用户输入注册邮箱
2. 发送密码重置验证码（有效期 15 分钟）
3. 用户输入验证码和新密码
4. 校验验证码正确性
5. 校验新密码符合强度要求
6. 更新密码并强制所有设备重新登录

**密码策略配置：**

租户管理员可配置以下策略：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `password_min_length` | 8 | 最小长度 |
| `password_strength_level` | MEDIUM | 强度等级 |
| `password_history_count` | 5 | 保留历史密码数量 |
| `password_max_age_days` | 90 | 密码有效期（天） |
| `password_expire_remind_days` | 7 | 过期前提醒天数 |
| `lockout_threshold` | 5 | 锁定阈值（错误次数） |
| `lockout_duration_minutes` | 15 | 锁定时长（分钟） |

**异常情况：**

| 异常场景 | 系统处理 |
|----------|----------|
| 密码强度不足 | 提示具体不满足的规则 |
| 密码与历史重复 | 提示「不能使用最近 N 次使用过的密码」 |
| 密码已过期 | 强制跳转修改密码页面 |
| 账号被锁定 | 提示锁定剩余时长 |
| 验证码错误/过期 | 提示错误信息，允许重新获取 |

**API 接口：**

```
POST   /api/v1/auth/password/change      # 修改密码
POST   /api/v1/auth/password/reset       # 重置密码（忘记密码）
POST   /api/v1/auth/password/verify      # 验证当前密码（敏感操作前）
GET    /api/v1/auth/password/expiry      # 查询密码过期状态
POST   /api/v1/auth/password/unlock      # 申请解锁（邮箱验证）
```

**数据库设计：**

**用户密码历史表（user_password_history）**

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| id | BIGINT | 是 | 主键 | 1001 |
| user_id | BIGINT | 是 | 用户 ID | 2001 |
| password_hash | VARCHAR(255) | 是 | 密码哈希（bcrypt/argon2） | $2a$10$xxxx... |
| created_at | DATETIME | - | 创建时间 | 2026-03-28 10:00:00 |

**索引**：`idx_user` (user_id, created_at)

---

**密码错误尝试记录表（password_login_attempts）**

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| id | BIGINT | 是 | 主键 | 2001 |
| user_id | BIGINT | 是 | 用户 ID | 2001 |
| attempt_time | DATETIME | 是 | 尝试时间 | 2026-03-28 10:00:00 |
| ip_address | VARCHAR(45) | 否 | IP 地址 | 192.168.1.100 |
| success | BOOLEAN | - | 是否成功 | true/false |

**索引**：`idx_user_time` (user_id, attempt_time)

---

**密码重置验证码表（password_reset_codes）**

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| id | BIGINT | 是 | 主键 | 3001 |
| user_id | BIGINT | 是 | 用户 ID | 2001 |
| code | VARCHAR(6) | 是 | 验证码 | 123456 |
| expires_at | DATETIME | 是 | 过期时间 | 2026-03-28 10:15:00 |
| used | BOOLEAN | - | 是否已使用 | true/false |
| created_at | DATETIME | - | 创建时间 | 2026-03-28 10:00:00 |

**索引**：`idx_user_code` (user_id, code)

**安全策略：**

| 策略 | 说明 |
|------|------|
| **加密存储** | 使用 bcrypt 或 argon2 算法 |
| **传输加密** | 全程 HTTPS，前端可加密后传输 |
| **不返回明文** | 任何场景不返回密码明文 |
| **失败模糊提示** | 不提示具体是用户不存在还是密码错误 |
| **密码修改通知** | 密码修改后发送邮件/短信通知 |
| **强制重新登录** | 密码修改后所有设备需要重新登录 |

**验收标准：**

- [ ] 弱密码被正确拒绝
- [ ] 历史密码不能重复使用
- [ ] 密码过期后强制修改
- [ ] 连续错误后账号正确锁定
- [ ] 密码重置验证码有效
- [ ] 密码修改后所有 Token 失效
- [ ] 密码策略可按租户配置
