# REQ-006 权限分配功能

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **估时** | 3 人天 |
| **关联用户故事** | US-021, US-022, US-023 |

**背景：** 需要将角色权限分配给用户，实现访问控制。

**目标：**

- 支持用户 - 角色关联
- 支持批量分配角色
- 支持查看用户最终权限
- 权限变更实时生效
- v1 不支持用户直接授予权限

**功能描述：**

1. 租户管理员可为用户分配角色
2. 支持批量选择用户，批量分配角色
3. 支持移除用户角色
4. 分配角色时可选指定应用范围（`app_code`）；未指定时表示租户级生效
5. 用户可拥有多个角色，最终权限取所有有效角色权限并集
6. 用户最终权限仅由角色决定，不存在用户直接权限、用户组权限和拒绝策略
7. 可查看用户的最终权限列表
8. 权限变更记录审计日志

**异常情况：**

| 异常场景 | 系统处理 |
|----------|----------|
| 用户不存在 | 提示"用户不存在" |
| 角色不存在 | 提示"角色不存在" |
| 重复分配同一角色 | 提示"用户已拥有该角色" |
| 指定应用不存在 | 提示"应用不存在" |
| 角色与应用不匹配 | 提示"角色不能在该应用范围内生效" |

**验收标准：**

- [ ] 可分配角色给用户
- [ ] 支持批量分配
- [ ] 用户可拥有多个角色
- [ ] 权限取并集生效
- [ ] 不支持用户直接权限和用户组权限
- [ ] 权限变更实时生效
- [ ] 审计日志完整记录
