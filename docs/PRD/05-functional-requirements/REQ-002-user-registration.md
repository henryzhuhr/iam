# REQ-002 用户注册与激活功能

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **估时** | 2 人天 |
| **关联用户故事** | US-003, US-004 |

**背景：** IAM 当前采用单租户用户模型，一个用户只能属于一个租户。为避免匿名注册后缺少租户归属、邮箱唯一性和权限初始化问题，首版本注册能力收敛为**邀请制注册/账号激活**，不支持脱离租户上下文的匿名开放注册。

**目标：**

- 支持基于邀请链接完成账号激活
- 支持密码强度校验和首登密码设置
- 保持用户与租户的一对一归属关系
- 支持租户管理员邀请新用户加入租户
- 目标注册转化率 > 80%

**功能范围：**

| 模式 | 是否纳入 v1 | 说明 |
|------|--------------|------|
| 邀请制注册 | 是 | 唯一注册入口，必须携带有效邀请链接或邀请令牌 |
| 匿名开放注册 | 否 | 不纳入当前版本，后续如需支持应拆分为独立需求，明确租户创建与归属规则 |

**主流程：**

1. 租户管理员或平台初始化流程向指定邮箱发送邀请链接，邀请中必须包含 `tenant_id` 和受邀邮箱上下文
2. 用户打开注册链接后，系统校验邀请令牌是否存在、是否过期、是否已被使用
3. 校验通过后，注册页展示租户名称和受邀邮箱，邮箱不可修改
4. 用户设置登录密码并补充基础资料
5. 系统按租户维度校验邮箱唯一性，规则为同租户不可重复、跨租户允许重复
6. 系统创建用户账号并将邀请状态更新为“已接受”
7. 注册成功后自动登录，返回 `Access Token + Refresh Token`，并跳转到首登完善信息页

**业务规则：**

1. 注册前必须已存在租户，注册本身不负责创建租户
2. 一个邀请只能使用一次，默认有效期 72 小时
3. 邀请必须绑定租户和邮箱，不能用于切换到其他租户
4. 同一邮箱可存在于不同租户下，不视为冲突
5. 同一租户下邮箱重复时拒绝注册
6. 租户被冻结或过期时，邀请不可继续激活

**异常情况：**

| 异常场景 | 系统处理 |
|----------|----------|
| 邀请令牌不存在或非法 | 提示"邀请链接无效，请联系管理员重新发送" |
| 邀请已过期 | 提示"邀请已过期，请联系管理员重新邀请" |
| 邀请已使用/已撤销 | 提示"邀请不可用，请联系管理员" |
| 受邀租户已冻结 | 提示"当前租户不可用，请联系平台管理员" |
| 密码强度不足 | 提示"密码需至少 8 位，包含大小写字母和数字" |
| 同租户下邮箱已存在 | 提示"该邮箱已在当前租户中使用" |
| 同邮箱在其他租户已存在 | 允许继续注册，不视为冲突 |

**接口约束：**

```text
POST /api/v1/auth/register    # 基于邀请令牌完成注册/激活
```

**数据设计补充：**

注册流程需要单独维护邀请记录，建议引入 `user_invitations` 表，至少包含以下字段：

- `tenant_id`：目标租户
- `email`：受邀邮箱
- `token_hash`：邀请令牌哈希
- `status`：待接受 / 已接受 / 已过期 / 已撤销
- `expires_at`：邀请过期时间
- `accepted_at`：接受时间

**验收标准：**

- [ ] 有效邀请链接 + 合规密码可完成注册
- [ ] 密码强度校验生效
- [ ] 过期、撤销、已使用邀请会被正确拒绝
- [ ] 同租户邮箱重复会被拦截，跨租户重复允许通过
- [ ] 注册成功后自动登录并绑定到邀请指定租户
