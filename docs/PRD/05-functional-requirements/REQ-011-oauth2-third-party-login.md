# REQ-011 OAuth2 第三方登录

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **估时** | 5 人天 |
| **关联用户故事** | US-015 |

**背景：** 用户希望使用已有的第三方账号（如 GitHub、Google、钉钉）快速登录系统，减少注册成本，提升登录体验。

**目标：**

- 支持 GitHub OAuth2 登录
- 支持邮箱自动绑定/注册
- 支持第三方登录与本地账号绑定
- 目标第三方登录转化率 > 70%

**功能描述：**

1. 用户访问登录页面，选择「使用 GitHub 登录」
2. 系统重定向到 GitHub OAuth2 授权页面
3. 用户授权后，GitHub 回调并返回 Authorization Code
4. 系统使用 Code 换取 Access Token，获取用户信息（邮箱、昵称、头像）
5. 根据邮箱匹配本地用户：
   - 邮箱已存在：直接登录成功
   - 邮箱不存在：自动创建用户并登录
6. 支持用户手动绑定/解绑第三方账号（在个人设置中）
7. 支持租户管理员配置允许/禁止的第三方登录方式

**OAuth2 流程：**

```
用户 → 点击 GitHub 登录 → 重定向到 GitHub 授权页
     → 用户同意授权 → GitHub 回调 /callback?code=xxx
     → 后端用 code 换取 access_token → 获取用户信息
     → 匹配/创建本地用户 → 颁发 JWT Token → 登录成功
```

**异常情况：**

| 异常场景 | 系统处理 |
|----------|----------|
| GitHub 授权失败 | 提示「授权失败，请重试」 |
| GitHub 接口超时 | 提示「获取用户信息超时，请重试」 |
| 邮箱已被绑定 | 提示「该 GitHub 账号已绑定其他账号，请先解绑」 |
| GitHub 邮箱未验证 | 拒绝登录，提示「请使用已验证的邮箱」 |
| OAuth2 回调参数缺失 | 提示「授权回调参数异常」 |

**第三方登录配置：**

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| GitHub Client ID | OAuth2 应用标识 | `Iv1.xxxxxxxxxxxxxxxx` |
| GitHub Client Secret | OAuth2 应用密钥 | `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` |
| Redirect URI | 回调地址 | `https://iam.example.com/auth/github/callback` |
| Scopes | 申请权限范围 | `read:user,user:email` |

**安全策略：**

| 策略 | 说明 |
|------|------|
| **邮箱验证** | 必须使用 GitHub 已验证的邮箱 |
| **账号绑定** | 支持一个本地账号绑定多个第三方账号 |
| **解绑限制** | 如设置为唯一登录方式，禁止解绑 |
| **登录日志** | 记录第三方登录来源、IP、时间 |

**验收标准：**

- [ ] GitHub 账号可成功授权登录
- [ ] 邮箱自动匹配/注册逻辑正确
- [ ] 支持手动绑定/解绑第三方账号
- [ ] 第三方登录配置可管理
- [ ] 登录日志完整记录
