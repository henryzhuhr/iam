# Go 代码审查清单 04: 语言细节与数据结构

本文件聚焦变量作用域、控制流、结构体、slice、map、序列化和时间语义。这里最容易出现的问题，是语言细节表面看着没错，实际却改变了行为、性能或 API 语义。

## 04. 语言细节与数据结构

### 04.1 变量声明与作用域

- 变量应尽量靠近使用点，减少“值已经失效但作用域还很大”的情况。
- 要重点警惕会改变行为的遮蔽，尤其是外层 `err`、命名返回值、循环变量和锁保护状态；不是所有 shadowing 都值得当成问题，但容易造成误判的 shadowing 值得指出。
- 数值转换、长度计算和外部输入参与算术时，要关注溢出、截断、符号变化和单位不一致。

### 04.2 常量与字面量

- 需要提取常量的，通常是协议字段、状态码、位掩码、时间单位、重试次数、分页上限等具有业务语义的值。
- 对只在局部使用、含义非常直接的小数字，不必机械提炼成常量。

### 04.3 for / range / switch / goto

- 在闭包或 goroutine 中引用循环变量时，要结合目标 Go 版本和具体写法判断是否安全；旧版本语言语义或复用外层变量时，仍可能出现“所有迭代都读到同一个值”的问题。
- 遍历大结构体切片时，是否需要按下标访问取决于复制成本和热点程度，不应默认视为 bug。
- `switch` 不必机械要求 `default`；更重要的是非法值、未来扩展值和不可能分支是否被清楚处理。
- `goto` 很少需要，但也不是绝对禁用；只有在它让控制流更难验证时才值得指出。

反例：

```go
for _, v := range ints {
	go func() {
		fmt.Println(v)
	}()
}
```

在旧语义或复用外层变量场景下更稳妥的写法：

```go
for _, v := range ints {
	v := v
	go func() {
		fmt.Println(v)
	}()
}
```

### 04.4 结构体

- 结构体拷贝是否安全，取决于字段里是否有引用类型、锁、缓存句柄或其他共享状态。
- 使用带字段名的字面量有助于避免位置参数写错，尤其是在跨包、字段较多或字段顺序可能演进时；对极小且稳定的局部结构体，不必一律强制。
- 字段顺序的内存对齐优化只有在大对象、热点路径或大量实例驻留时才值得重点关注。

### 04.5 slice

- 下标访问和切片表达式首先是正确性问题，要确认不会越界。
- 预分配容量通常是性能建议，只有在大循环、热点路径或明显重复扩容时才值得强调。
- `nil` slice 和空 slice 都可能合理；关键是 API 语义、序列化结果和调用方预期是否一致。
- `copy`、append 共享底层数组、子切片长期持有大数组，这些都要按对象生命周期和数据所有权判断。

反例：

```go
src := []int{0, 1, 2}
var dst []int
copy(dst, src)
```

优秀示例：

```go
src := []int{0, 1, 2}
dst := make([]int, len(src))
copy(dst, src)
```

### 04.6 map

- 向 nil map 写入会 panic，这是明确的正确性问题。
- 只有在需要区分“key 不存在”和“value 恰好是零值”时，才必须使用 `v, ok := m[k]`。
- map 是否预分配容量、是否用字面量初始化，通常是可读性或性能建议，不应机械上升为硬规则。
- 共享 map 的并发读写是不安全的，这类问题优先归到并发审查。

### 04.7 序列化与默认值

- tag 名称、`omitempty`、字段导出性、零值含义、指针字段和版本兼容性都可能改变外部行为，要重点检查。
- “看起来有默认值、实际不会生效”是 review 里很常见的真实问题，例如把业务默认值写在反序列化之后但又被零值覆盖。

优秀示例：

```go
type Response struct {
	Message string `json:"message"`
}
```

### 04.8 time

- 时间点、持续时间、超时、时区和闰秒/单调时钟语义要分清；能用 `time.Duration`、`time.Time` 表达的，不要退化成裸整数再靠注释补语义。
- 涉及外部 IO、重试和后台任务时，要检查 timeout、deadline 和 ticker 生命周期是否与业务预期一致。

优秀示例：

```go
ctx, cancel := context.WithTimeout(parent, 100*time.Millisecond)
defer cancel()
```
